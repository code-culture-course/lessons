# Примеры ввода/вывода

Эта директория содержит практические примеры, демонстрирующие концепции ввода/вывода в Python.

## Файлы примеров

### Консольный ввод/вывод

**[01_basic_input.py](01_basic_input.py)** — Базовый ввод и вывод
- Использование `print()` для отображения сообщений
- Использование `input()` для чтения пользовательского ввода
- Конвертация строк в числа
- Простая программа создания персонажа

**[02_input_validation.py](02_input_validation.py)** — Валидация ввода
- Обработка некорректного числового ввода
- Использование try/except для обработки ошибок
- Создание вспомогательных функций для безопасного ввода
- Запрос ввода до получения валидных данных

**[03_simple_menu.py](03_simple_menu.py)** — Простое меню
- Построение интерактивного меню с циклом
- Обработка выбора пользователя
- Выход из циклов
- Основа для CLI приложений

### Операции с файлами

**[04_write_file.py](04_write_file.py)** — Запись в файл
- Открытие файла для записи
- Запись текста в файл
- Корректное закрытие файлов
- Понимание режима `"w"` (перезапись)

**[05_read_file.py](05_read_file.py)** — Чтение из файла
- Открытие файла для чтения
- Чтение всего содержимого файла
- Построчное чтение
- Обработка отсутствующих файлов

**[06_append_file.py](06_append_file.py)** — Добавление в файл
- Использование режима `"a"` для добавления
- Добавление содержимого без стирания
- Разница между режимами записи и добавления

### JSON

**[07_save_json.py](07_save_json.py)** — Сохранение данных в JSON
- Конвертация словарей Python в JSON
- Запись JSON в файл
- Почему JSON полезен для структурированных данных
- Читаемое для человека форматирование

**[08_load_json.py](08_load_json.py)** — Загрузка данных из JSON
- Чтение JSON из файла
- Конвертация JSON обратно в объекты Python
- Работа с загруженными данными
- Обработка ошибок JSON

**[09_json_persistence.py](09_json_persistence.py)** — Полный цикл сохранения JSON
- Загрузка данных если они существуют
- Использование данных по умолчанию если их нет
- Изменение данных
- Сохранение обратно в файл
- Полная демонстрация постоянства

### Мини-проект

**[10_mini_project/](10_mini_project/)** — Пошаговый менеджер задач CLI

Полный мини-проект, показывающий как построить реальное приложение:
- **step1_menu.py** — Базовая структура меню
- **step2_tasks_in_memory.py** — Хранение задач в памяти
- **step3_save_to_file.py** — Добавление сохранения в файл
- **step4_json_persistence.py** — Использование JSON для лучшей структуры
- **step5_error_handling.py** — Добавление надёжной обработки ошибок

См. [10_mini_project/README.md](10_mini_project/README.md) для подробностей.

## Запуск примеров

Запустите любой пример с помощью Python:

```bash
python 01_basic_input.py
python 02_input_validation.py
# ... и т.д.
```

## Путь обучения

Для лучшего опыта обучения:

1. **Начните с консольного ввода/вывода** (01-03)
   - Узнайте как программы взаимодействуют с пользователями
   - Обретите уверенность с вводом/выводом

2. **Переходите к операциям с файлами** (04-06)
   - Поймите постоянство
   - Изучите режимы файлов

3. **Прогрессируйте к JSON** (07-09)
   - Увидьте почему структурированные данные важны
   - Изучите основы JSON

4. **Завершите мини-проект** (10)
   - Примените все концепции вместе
   - Постройте реальное приложение шаг за шагом

## Ключевые концепции

- **Консольный ввод/вывод**: Прямое взаимодействие с пользователем
- **Файлы**: Данные, которые сохраняются между запусками
- **JSON**: Структурированный, переносимый формат данных
- **Обработка ошибок**: Создание надёжных программ
- **Разделение ответственности**: Организация кода для ясности
